这段代码说明了 PLONK 的核心作用：证明某些计算关系成立，但不暴露参与计算的私密数据。

  在这个例子里，证明者想向验证者证明自己知道一些私密值：

  a, b, d, e

  并且这些值满足下面的规则：

  1. a < 2^6
  2. b < 2^4
  3. a + b + 42 = c
  4. a * b + d = 42
  5. e * JubJub::GENERATOR = f

  其中：

  c
  f

  是公开输入，验证者可以看到；而：

  a, b, d, e

  是私密 witness，验证者看不到。

  所以这段代码表达的是：

  > 我知道一些隐藏值 a, b, d, e，它们满足这些约束，并且会得到公开结果 c 和 f。我可以证明
  > 这件事是真的，但不告诉你这些隐藏值是什么。

  PLONK 在这里的作用就是：

  - 把这些数学关系编译成约束电路；
  - 让 prover 生成一个零知识证明；
  - 让 verifier 只用 proof 和 public inputs 验证结论；
  - verifier 不需要重新知道或计算私密输入。

  一句话概括：PLONK 用来验证“某个计算过程确实正确执行了”，同时隐藏计算中的私密输入。
