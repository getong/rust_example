* 项目简介

这是一个使用 Rust 的 =tch= crate（PyTorch 的 Rust 绑定）的示例项目。

* 环境要求

- Rust (推荐使用最新稳定版)
- Python 3.x
- uv (Python 包管理器)
- 项目根目录 =.venv=

* 编译步骤

** 1. 创建虚拟环境并安装 Python 依赖

首先安装 uv（如果尚未安装）：

#+BEGIN_SRC bash
curl -LsSf https://astral.sh/uv/install.sh | sh
#+END_SRC

使用 uv 创建虚拟环境并安装 PyTorch：

#+BEGIN_SRC bash
uv venv .venv
source .venv/bin/activate
uv pip install -r requirements.txt
#+END_SRC

验证安装：

#+BEGIN_SRC bash
python -W ignore -c "import torch; print(torch.__version__.split('+')[0])"
#+END_SRC

预期输出：

#+BEGIN_EXAMPLE
2.11.0
#+END_EXAMPLE

** 2. 设置编译环境变量

确保激活虚拟环境，并设置 =LIBTORCH_USE_PYTORCH= 环境变量：

#+BEGIN_SRC bash
source .venv/bin/activate
export LIBTORCH_USE_PYTORCH=1
#+END_SRC

** 3. 设置运行时库路径

设置 =DYLD_LIBRARY_PATH= 环境变量以便运行时能找到 PyTorch 动态库：

#+BEGIN_SRC bash
TORCH_LIB=$(python -W ignore -c "import torch; import os; print(os.path.join(os.path.dirname(torch.__file__), 'lib'))")
export DYLD_LIBRARY_PATH=$TORCH_LIB:$DYLD_LIBRARY_PATH
#+END_SRC

** 4. 编译项目

#+BEGIN_SRC bash
cargo build --bin tch_basic --release
#+END_SRC

** 5. 运行程序

#+BEGIN_SRC bash
cargo run --bin tch_basic --release
#+END_SRC

预期输出：

#+BEGIN_EXAMPLE
  6
  2
  8
  2
 10
[ CPUIntType{5} ]
#+END_EXAMPLE

* 一键编译和运行脚本

脚本只负责激活现有虚拟环境、设置变量并执行 cargo：

#+BEGIN_SRC bash
#!/bin/bash
# 保存为 build_and_run.sh
# 在项目根目录执行此脚本

# 激活虚拟环境
source .venv/bin/activate
export LIBTORCH_USE_PYTORCH=1

# 设置运行时库路径
TORCH_LIB=$(python -W ignore -c "import torch; import os; print(os.path.join(os.path.dirname(torch.__file__), 'lib'))")
export DYLD_LIBRARY_PATH=$TORCH_LIB:$DYLD_LIBRARY_PATH

# 编译并运行
cargo build --bin tch_basic --release
cargo run --bin tch_basic --release
#+END_SRC

使用方式：

#+BEGIN_SRC bash
chmod +x build_and_run.sh
./build_and_run.sh
#+END_SRC

* 常见问题

** 错误：Cannot find a libtorch install

*原因*：未找到 libtorch 安装。

*解决方案*：
1. 创建并激活虚拟环境：=uv venv .venv && source .venv/bin/activate=
2. 安装 Python 依赖：=uv pip install -r requirements.txt=
3. 设置环境变量：=export LIBTORCH_USE_PYTORCH=1=

** 错误：ModuleNotFoundError: No module named 'torch'

*原因*：当前 =.venv= 中未安装 PyTorch，或虚拟环境未激活。

*解决方案*：
- 确保已激活虚拟环境：=source .venv/bin/activate=
- 确保虚拟环境中已安装依赖：=uv pip install -r requirements.txt=
- 验证安装：=python -W ignore -c "import torch; print(torch.__version__.split('+')[0])"=

** 错误：PyTorch 版本不匹配

*原因*：=tch 0.24.0= 需要 PyTorch =2.11.0=。

*解决方案*：
- 使用 =requirements.txt= 中锁定的版本：=uv pip install -r requirements.txt=
- 验证版本：=python -W ignore -c "import torch; print(torch.__version__.split('+')[0])"=

** 错误：Library not loaded: @rpath/libtorch_cpu.dylib

*原因*：运行时找不到 PyTorch 的动态库。

*解决方案*：
1. 确保已激活虚拟环境：=source .venv/bin/activate=
2. 设置运行时库路径：
   #+BEGIN_SRC bash
   TORCH_LIB=$(python -W ignore -c "import torch; import os; print(os.path.join(os.path.dirname(torch.__file__), 'lib'))")
   export DYLD_LIBRARY_PATH=$TORCH_LIB:$DYLD_LIBRARY_PATH
   #+END_SRC
3. 然后运行程序：=cargo run --bin tch_basic --release=
