#+title: Async GraphQL Workspace Example

* 项目说明

这个 workspace 包含 4 个基于 ~async-graphql~ 和 ~axum~ 的 GraphQL 示例：

| crate             | 说明                                               | HTTP 端点 | WebSocket 端点 |
|-------------------+----------------------------------------------------+----------+----------------|
| ~starwars~          | Star Wars 查询示例，包含接口、枚举和分页连接           | ~/~        | 无             |
| ~subscription~      | 图书查询、创建、删除和订阅示例                         | ~/~        | ~/ws~            |
| ~token_from_header~ | 从 HTTP Header 或 WebSocket init payload 读取 token | ~/~        | ~/ws~            |
| ~upload~            | GraphQL 文件上传示例                                 | ~/~        | 无             |

所有示例默认监听 ~127.0.0.1:8000~。同一时间只能运行一个示例；切换示例前请先停止当前进程。

* 环境要求

- Rust 2024 edition toolchain
- ~curl~

可选：

- 浏览器访问 GraphiQL：[[http://localhost:8000]]

* 启动示例

在 workspace 根目录运行其中一个命令：

#+begin_src sh
cargo run -p starwars
#+end_src

#+begin_src sh
cargo run -p subscription
#+end_src

#+begin_src sh
cargo run -p token_from_header
#+end_src

#+begin_src sh
cargo run -p upload
#+end_src

下面的 ~curl~ 示例都假设服务已经运行在 ~http://127.0.0.1:8000/~。

* 使用 curl 测试 starwars

启动：

#+begin_src sh
cargo run -p starwars
#+end_src

查询默认英雄：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  --data '{"query":"query { hero { id name appearsIn friends { id name } ... on Human { homePlanet } ... on Droid { primaryFunction } } }"}' \
  | jq
#+end_src

按电影查询英雄：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  --data '{"query":"query ($episode: Episode!) { hero(episode: $episode) { id name appearsIn } }","variables":{"episode":"EMPIRE"}}' \
  | jq
#+end_src

按 ID 查询 Human：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  --data '{"query":"query { human(id: \"1000\") { id name homePlanet friends { id name } } }"}' \
  | jq
#+end_src

分页查询 Humans：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  --data '{"query":"query { humans(first: 2) { edges { cursor node { id name appearsIn } } pageInfo { hasNextPage hasPreviousPage } } }"}' \
  | jq
#+end_src

查询 Droids：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  --data '{"query":"query { droids(first: 10) { edges { cursor node { id name primaryFunction } } } }"}' \
  | jq
#+end_src

* 使用 curl 测试 subscription

启动：

#+begin_src sh
cargo run -p subscription
#+end_src

查询图书列表：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  --data '{"query":"query { books { id name author } }"}' \
  | jq
#+end_src

创建图书：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  --data '{"query":"mutation ($name: String!, $author: String!) { createBook(name: $name, author: $author) }","variables":{"name":"The Rust Programming Language","author":"Steve Klabnik and Carol Nichols"}}' \
  | jq
#+end_src

再次查询图书列表：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  --data '{"query":"query { books { id name author } }"}' \
  | jq
#+end_src

删除图书。这里的 ~id~ 使用创建图书返回的值，例如 ~"0"~：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  --data '{"query":"mutation ($id: ID!) { deleteBook(id: $id) }","variables":{"id":"0"}}' \
  | jq
#+end_src

说明：这个示例的订阅端点是 WebSocket ~/ws~，GraphQL subscription 需要按 ~graphql-transport-ws~ 或旧 GraphQL WebSocket 协议发送连接初始化和订阅消息。普通 HTTP ~curl~ 适合测试 query/mutation，不适合完整测试 subscription 流；可以直接打开 GraphiQL 测试：

#+begin_src graphql
subscription {
  interval(n: 2)
}
#+end_src

#+begin_src graphql
subscription {
  books {
    mutationType
    id
    book {
      id
      name
      author
    }
  }
}
#+end_src

* 使用 curl 测试 token_from_header

启动：

#+begin_src sh
cargo run -p token_from_header
#+end_src

不带 Token Header 时查询当前 token，返回 ~null~：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  --data '{"query":"query { currentToken }"}' \
  | jq
#+end_src

带 Token Header 查询当前 token：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  -H 'Token: 123456' \
  --data '{"query":"query { currentToken }"}' \
  | jq
#+end_src

说明：~values~ 是 WebSocket subscription，需要在连接初始化 payload 中传入 ~{"token":"123456"}~。普通 HTTP ~curl~ 无法完整执行这个订阅流；可以在 GraphiQL 中配置 subscription connection params 后测试：

#+begin_src json
{
  "token": "123456"
}
#+end_src

然后运行：

#+begin_src graphql
subscription {
  values
}
#+end_src

* 使用 curl 测试 upload

启动：

#+begin_src sh
cargo run -p upload
#+end_src

查询已上传文件列表：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  --data '{"query":"query { uploads { id url } }"}' \
  | jq
#+end_src

单文件上传。示例会把上传文件名保存为 ~url~ 字段：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -F 'operations={"query":"mutation ($file: Upload!) { singleUpload(file: $file) { id url } }","variables":{"file":null}}' \
  -F 'map={"0":["variables.file"]}' \
  -F '0=@Cargo.toml' \
  | jq
#+end_src

多文件上传：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -F 'operations={"query":"mutation ($files: [Upload!]!) { multipleUpload(files: $files) { id url } }","variables":{"files":[null,null]}}' \
  -F 'map={"0":["variables.files.0"],"1":["variables.files.1"]}' \
  -F '0=@Cargo.toml' \
  -F '1=@starwars/Cargo.toml' \
  | jq
#+end_src

上传后再次查询：

#+begin_src sh
curl -sS 'http://127.0.0.1:8000/' \
  -H 'content-type: application/json' \
  --data '{"query":"query { uploads { id url } }"}' \
  | jq
#+end_src

* 常见问题

** 端口被占用

所有示例都绑定 ~127.0.0.1:8000~。如果启动时报 ~Address already in use~，先停止正在运行的示例，再启动新的 crate。

** GraphQL 字段命名

Rust 方法名会映射成 GraphQL 的 camelCase 字段，例如：

| Rust 方法         | GraphQL 字段    |
|------------------+-----------------|
| ~appears_in~       | ~appearsIn~       |
| ~home_planet~      | ~homePlanet~      |
| ~primary_function~ | ~primaryFunction~ |
| ~create_book~      | ~createBook~      |
| ~single_upload~    | ~singleUpload~    |

** 同时测试多个示例

当前代码中端口是写死的 ~8000~。如果需要同时运行多个示例，需要分别修改对应 crate 的 ~src/main.rs~ 监听端口。
