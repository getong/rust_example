* libp2p openraft multi-raft rocksdb

** usage

Raft groups: `users`, `orders`, `products`. Default group is `users`.
Use `group_id` in HTTP bodies, `--group` in CLI tools, or `?group=` on `/cluster`.

Three-node local runs enable =console-subscriber= for tokio-console. Each node
binds a different console port to avoid conflicts:

#+begin_src shell
./run-3nodes.sh

tokio-console http://127.0.0.1:6669 # node1
tokio-console http://127.0.0.1:6670 # node2
tokio-console http://127.0.0.1:6671 # node3
#+end_src

Override the defaults with =NODE1_TOKIO_CONSOLE_BIND=,
=NODE2_TOKIO_CONSOLE_BIND=, or =NODE3_TOKIO_CONSOLE_BIND= when needed.

#+begin_src shell
# set (default group: users)
curl -X POST http://127.0.0.1:3001/write \
-H 'content-type: application/json' \
-d '{"key":"user:1","value":"Alice"}'

# set into orders group
curl -X POST http://127.0.0.1:3001/write \
-H 'content-type: application/json' \
-d '{"group_id":"orders","key":"order:1001","value":"pending"}'

# update products group
curl -X POST http://127.0.0.1:3002/update \
-H 'content-type: application/json' \
-d '{"group_id":"products","key":"product:A","value":"Widget"}'

# delete from users group
curl -X POST http://127.0.0.1:3002/delete \
-H 'content-type: application/json' \
-d '{"group_id":"users","key":"user:1"}'

# get cluster info (default group)
curl http://127.0.0.1:3001/cluster

# get cluster info for orders group
curl "http://127.0.0.1:3001/cluster?group=orders"

# list OpenRaft members for a group
curl http://127.0.0.1:3001/openraft/nodes
curl "http://127.0.0.1:3001/openraft/nodes?group=apalis"

# list libp2p known peers and their OpenRaft role, when any
curl http://127.0.0.1:3001/libp2p/nodes
curl "http://127.0.0.1:3001/libp2p/nodes?group=apalis"

# chat
curl -X POST http://127.0.0.1:3001/chat \
  -H 'content-type: application/json' \
  -d '{"text":"hello from http","from":"node1"}'

# publish this node's latest OpenRaft snapshot through libp2p gossipsub
# partial messages; receivers install it through OpenRaft APIs
curl -X POST http://127.0.0.1:3001/sync/snapshot \
  -H 'content-type: application/json' \
  -d '{"group_id":"users"}'

# enqueue an apalis email task backed by the apalis raft control plane
curl -X POST http://127.0.0.1:3001/apalis/email \
  -H 'content-type: application/json' \
  -d '{"to":"test@example.com"}'

# list registered external workers and task state
curl http://127.0.0.1:3001/apalis/workers
curl http://127.0.0.1:3001/apalis/tasks

# safely remove an OpenRaft control node before shutting that node down
# send this to the current leader's HTTP port; omit group_id to remove from all groups
curl -X POST http://127.0.0.1:3001/openraft/membership/remove \
  -H 'content-type: application/json' \
  -d '{"node_id":"<node-id-to-remove>"}'

# or remove from a single raft group
curl -X POST http://127.0.0.1:3001/openraft/membership/remove \
  -H 'content-type: application/json' \
  -d '{"group_id":"users","node_id":"<node-id-to-remove>"}'
#+end_src

** apalis

Nodes start as libp2p workers by default. The node whose =--id= matches the
configured =--bootstrap-node= id creates the initial OpenRaft cluster. Other
nodes dial =--bootstrap-node= and ask the current OpenRaft leader to join;
automatic startup join stops when the control membership reaches
=--max-control-nodes=, which defaults to 3. Nodes accepted into OpenRaft
membership promote to control nodes, and later nodes stay as workers.

Tasks are stored in the =apalis= raft group under
=apalis:apalis:task:<task-id>=. Worker lease records are stored under
=apalis:apalis:worker:<node-id>=. The Raft leader schedules queued tasks to
active worker leases, writes the assigned worker and lease epoch through
OpenRaft, and the target worker executes the task locally with apalis. Final
task state is acknowledged back through the Raft-backed KV protocol.

Start the control plane:

#+begin_src shell
./run-3nodes.sh
#+end_src

Start one or more external workers after copying the printed =DB_ROOT=:

#+begin_src shell
DB_ROOT=/tmp/openraft_libp2p_cluster_demo/<run-id> WORKER_INDEX=1 ./run-worker.sh
DB_ROOT=/tmp/openraft_libp2p_cluster_demo/<run-id> WORKER_INDEX=2 ./run-worker.sh
#+end_src

Or fetch the current control-plane information from =/cluster= and start four
workers at once:

#+begin_src shell
DB_ROOT=/tmp/openraft_libp2p_cluster_demo/<run-id> ./join-4workers.sh

# optional overrides
CLUSTER_HTTP=http://127.0.0.1:3002/cluster WORKER_COUNT=4 \
  DB_ROOT=/tmp/openraft_libp2p_cluster_demo/<run-id> ./join-4workers.sh
#+end_src

Worker libp2p listeners bind to =/ip4/0.0.0.0/tcp/<port>/wss= by default so
the LAN addresses advertised by mDNS are dialable by other peers. Worker HTTP
ports stay on =127.0.0.1= by default.

Workers can also accept task submissions on their own HTTP ports; they forward
the writes to the control plane:

#+begin_src shell
curl -X POST http://127.0.0.1:3101/apalis/email \
  -H 'content-type: application/json' \
  -d '{"to":"worker-submit@example.com"}'
#+end_src

Promote a running worker into OpenRaft membership by submitting the request to
the current leader:

#+begin_src shell
curl -X POST http://127.0.0.1:3001/openraft/membership/add \
  -H 'content-type: application/json' \
  -d '{"node_id":"<worker-peer-id>","addr":"/ip4/127.0.0.1/tcp/4101/wss/p2p/<worker-peer-id>"}'
#+end_src

** cli examples

#+begin_src shell
# write into users group
cargo run -p openraft_libp2p_cluster --bin olpc-kv -- \
  --addr /ip4/127.0.0.1/tcp/4001/p2p/<peerid1> \
  --group users \
  set --key user:2 --value Bob

# read from orders group
cargo run -p openraft_libp2p_cluster --bin olpc-kv -- \
  --addr /ip4/127.0.0.1/tcp/4001/p2p/<peerid1> \
  --group orders \
  get --key order:1001

# query metrics for products group
cargo run -p openraft_libp2p_cluster --bin olpc-status -- \
  --addr /ip4/127.0.0.1/tcp/4001/p2p/<peerid1> \
  --group products
#+end_src

** websocket

This project enables a websocket transport alongside TCP and QUIC. Use /ws
multiaddrs for listen and peer addresses.

#+begin_src shell
# listen with websocket
cargo run -p openraft_libp2p_cluster --bin openraft_libp2p_cluster -- \
  --id <peerid1> \
  --listen /ip4/0.0.0.0/tcp/4001/ws \
  --http 0.0.0.0:3001 \
  --db /tmp/openraft-node1 \
  --bootstrap-node <peerid1>=/ip4/127.0.0.1/tcp/4001/ws/p2p/<peerid1> \
  --advertise /ip4/127.0.0.1/tcp/4001/ws/p2p/<peerid1>

# join through the bootstrap node
cargo run -p openraft_libp2p_cluster --bin openraft_libp2p_cluster -- \
  --id <peerid2> \
  --listen /ip4/0.0.0.0/tcp/4002/ws \
  --http 0.0.0.0:3002 \
  --db /tmp/openraft-node2 \
  --bootstrap-node <peerid1>=/ip4/127.0.0.1/tcp/4001/ws/p2p/<peerid1> \
  --advertise /ip4/127.0.0.1/tcp/4002/ws/p2p/<peerid2>

# use websocket address in the kv client
cargo run -p openraft_libp2p_cluster --bin olpc-kv -- \
  --addr /ip4/127.0.0.1/tcp/4001/ws/p2p/<peerid1> \
  --group users \
  get --key counter

# use websocket address in the status client
cargo run -p openraft_libp2p_cluster --bin olpc-status -- \
  --addr /ip4/127.0.0.1/tcp/4001/ws/p2p/<peerid1>
#+end_src

Websocket limits (optional):

#+begin_src shell
# limit websocket frame size and redirects
--ws-max-data-size 10485760 --ws-max-redirects 2
#+end_src

Secure websocket (wss):

#+begin_src shell
# generate a local self-signed cert (DER)
./generate_wss_certs.sh ./certs localhost 127.0.0.1

# listen with wss using the generated cert
cargo run -p openraft_libp2p_cluster --bin openraft_libp2p_cluster -- \
  --id <peerid1> \
  --listen /ip4/0.0.0.0/tcp/4001/wss \
  --http 0.0.0.0:3001 \
  --db /tmp/openraft-node1 \
  --ws-tls-key ./certs/private.der \
  --ws-tls-cert ./certs/fullchain.der

# trust the self-signed cert when dialing
cargo run -p openraft_libp2p_cluster --bin olpc-kv -- \
  --addr /ip4/127.0.0.1/tcp/4001/wss/p2p/<peerid1> \
  --group users \
  --ws-tls-cert ./certs/fullchain.der \
  get --key counter
#+end_src

Notes:
- Websocket addresses use /ws (plain websocket). Secure websocket (/wss or /tls/ws)
  is not configured by default.
- You can keep using /tcp and /udp/quic-v1; websocket is an additional transport.
